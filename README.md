### Задание 1 На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия


Ответить в свободной форме.`

 `Различия master‑slave и master‑master репликации 
Master‑slave

Принцип: один главный сервер (master) + подчинённые (slave).
Запись: только на master.
Чтение: с любого сервера.
Плюсы: просто настроить, хорошо для разгрузки чтением, подходит для резервного копирования.
Минусы: при падении master запись останавливается; возможна задержка репликации.
Когда использовать: много чтений, мало записей; нужен простой резерв.
Master‑master

Принцип: все серверы равноправны — каждый и master, и slave.
Запись: на любой сервер.
Чтение: с любого сервера.
Плюсы: высокая доступность (система живёт при падении одного сервера), локальные записи в разных зонах.
Минусы: сложно настроить, возможны конфликты при одновременных изменениях.
Когда использовать: критична безотказность; распределённая инфраструктура с записями в разных локациях.
Ключевое отличие:

В master‑slave запись только на один сервер, в master‑master — на любой.
Master‑master устойчивее к отказам, но сложнее в настройке.`
